Community-maintained profiles

画像信息怎么维护？

职业、公开位置、语言信号这类字段不应该包装成确定事实。建议把它们作为开源仓库里的可审阅数据： 只有用户自己公开写过、或本人提交 PR 的信息才合并。

你可以编辑 profiles.json，以用户名为 key 添加 occupation、public_location、language_signal、inferred_region、confidence、links 和 note。 

{
  "Yoorkin": {
    "occupation": "Compiler / tooling developer",
    "public_location": "Unknown",
    "language_signal": "English / Chinese",
    "inferred_region": "Do not infer nationality from language alone",
    "confidence": "low",
    "links": ["https://github.com/Yoorkin"],
    "note": "Example profile annotation"
  }
}

Methodology

统计口径

  1. 读取公开统计接口，尽量兼容常见字段名：owner、username、created_at、upload_time 等。
  2. 按用户聚合包记录，取最早的包上传时间作为“首次贡献包”时间。
  3. 按 ISO 周分组，生成每周首次贡献用户数与活跃上传用户数。
  4. 职业、公开位置、语言信号不从包数据推断，只从 profiles.json 显示；地区线索必须保留置信度与来源。
